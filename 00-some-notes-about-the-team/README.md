# Welcome :)

We are so happy to have you onboard our Software wing of the team. I wanted to publish this as a way to show how we do things here, and hopefully a reference for the future, if necessary.

## Policies

### Team

- A project is done after it is commented/documented, and we feel comfortable open-sourcing it to the campus community
- Trying and failing (and sometimes falling flat on your face) is a part of the process, and we are very comfortable picking ourselves back up and learning from it.
- School and grades are more important than this job.
- Your mental health is **definitely** more important than this job
- If you need help or don't understand something, please ask for help. The door is always open for you.

### Department

- Students are not permitted to work from home
- Students may not work on the weekends

### University

- Students may work from 8AM until 6PM without additional approval (if you would like to work outside of this range, come talk to me)
- Student employees are expected to uphold the Honor Code, including dress and grooming standards
- Clock in/out on Workday, submit time at the end of your Friday shift, I approve it by Monday at Noon

## Processes

- A lot of our shared files will be on Box
- GitHub is where we keep all of our code
- Project managment stuff will be handled by GitHub Projects (See module 9)
- We work with feature branches, so create a branch for each issue/bugfix you're working on, name it something like #issue-number-Description-of-issue. When it's done, submit a Pull Request (see module 8) and then we'll go from there.
- Don't share API keys or Client Secrets or commit them to GitHub
- We move *fast*, so don't be afraid to ask for help or take initiative to try something new
